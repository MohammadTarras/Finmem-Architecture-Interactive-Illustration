System overview — TSLA, 2023

Three core modules

P

Profiling

Defines the agent character: professional background (EV/auto sector), and risk inclination (risk-seeking, risk-averse, or self-adaptive). Adapts when cumulative return crosses zero.

M

Memory

Four-layer database (short, mid, long, reflection). Each news article, filing, and reflection is embedded with OpenAI text-embedding-ada-002 (1536-d), scored, and ranked.

D

Decision-making

LLM (GPT-4-Turbo) receives Top-K memories from all layers + momentum signal + risk character → outputs BUY / SELL / HOLD with cited memory IDs.

Memory layers

Short-term

init imp: 50–90updated daily

10-Q quarterly filings · decay Q=90 days
init imp: 60–80updated quarterly
Long-term
10-K annual filings · decay Q=365 days
init imp: 80+updated yearly

init imp: 80cumulative

Training phase — building memory from labelled price data

What happens each training day

1

Receive market info

Date, price, filings (10-K/10-Q if available), list of news articles, and the future price diff (the label). From log: cur_date=2023-02-10 price=$196.89 future_diff=−2.25 news_count=11

2

Embed & store news

Each article is vectorized and inserted into the short-term FAISS index with initial importance drawn from a piecewise distribution (p1=0.8/p2=0.15/p3=0.05 → values 40/60/80, scaled by layer). Log shows IDs 0–10 with scores 50–90.

3

Retrieve Top-K memories

Query vector = embed("TSLA"). Cosine similarity search returns candidate IDs ranked by composite score. From log: ids=[8,3,1,0,6,7,9,4,10,2] by cosine, then re-ranked → returned=[0,1,2,3,5,8]

4

Training prompt → LLM

The agent is told the future price direction. It must explain why the price moved that way, citing memory IDs. This creates a reasoning summary stored as a reflection.

5

Decay + cleanup + jump

All stored memories get decayed: recency drops from 1.0 → 0.716 on day 1. Importance also drops (e.g. 70 → 64.4). Memories below thresholds are removed. Important ones "jump" to deeper layers.

What changes in testing

1

No future price label

Instead of a price diff, the agent receives momentum: cumulative log return over last M=7 days. This replaces the "ground truth" signal.

2

Risk character activates

If cumulative return ≥ 0 → risk-seeking mode. If < 0 → risk-averse mode. This is injected into the prompt dynamically.

3

All 4 memory layers queried

Top-10 from short, mid, long, and reflection layers. Total context can include 40 memory snippets. The LLM must weigh recent news vs quarterly vs annual vs its own past reasoning.

4

Output: BUY / SELL / HOLD + IDs

Guardrails AI validates the JSON schema. The cited IDs trigger access counter increments, boosting those memories' importance scores by +5.

5

Portfolio update + feedback

After a 7-day lookback window, the daily PnL accumulates. If cumulative profit is positive, cited memories get importance boosts. Memories slide toward deeper layers via "jump" thresholds.

Memory dynamics — decay, cleanup, and layer promotion

Scoring formula

γ = S_recency + S_relevancy + S_importance
Recency
S_rec = e^(−δ/Q)
δ = days since creation
Q = 14 (short) / 90 (mid) / 365 (long)
From log — day 1 decay

recency: 1.0 → 0.7165
(δ=1, Q=14: e^−1/14)

Importance
S_imp = v × α^δ
v = 40/60/80 (piecewise)
α = 0.9 (short) / 0.967 (mid) / 0.988 (long)
From log — day 1 decay

imp: 70 → 64.4 (×0.92)
imp: 50 → 46.0
imp: 90 → 82.8

Relevancy (cosine sim)
S_rel = (m_E · m_P) / (‖m_E‖ × ‖m_P‖)
m_E = memory embedding · m_P = query embedding ("TSLA" or full character prompt)
From log — cosine distances on 2023-02-10

Top by similarity: ID 8 (0.867), ID 3 (0.859), ID 1 (0.848), ID 0 (0.825)...
After importance re-rank → final order: [0,1,2,3,5,8]

Cleanup thresholds

Memory is deleted when:
recency < 0.05
OR
importance < 5

Layer promotion (jump)

Memories cited during decisions get access_counter += 1 → importance += 5.
Short (imp 0–60) →
Mid (imp 60–80) →
Long (imp 80+)
On jump: recency is reset to 1.0 to prevent rapid re-decay after promotion.
Retrieval mechanism — hybrid cosine + importance ranking

Two-stage retrieval (from log, day 1)

1

FAISS cosine search

Query = embed("TSLA") → 1536-d vector. FAISS returns top candidates by cosine similarity.

Log output

dists=[[0.867, 0.859, 0.848, 0.825, 0.821, 0.821, 0.820, 0.815, 0.809, 0.798]]
ids=[[8, 3, 1, 0, 6, 7, 9, 4, 10, 2]]

2

Importance re-ranking

Composite score = recency + relevancy + importance. Candidates are sorted by this compound score. Top-K are selected per layer.

Log output — re-ranked result

returned_ids=[0, 1, 2, 3, 5, 8] count=6
Note: ID 5 (imp=90) entered top-K despite lower cosine sim (0.798) because high importance boosts its composite score.

3

Composite score formula

compound = (importance × recency) + cosine_sim
Log shows: partial_score = important_score_recency_compound_score + cosine_offset
ID 5: imp=90, rec=1.0 → compound=1.9 (vs ID 9: imp=50, rec=1.0 → 1.5)

Access counter → importance boost

When the LLM cites memory ID 3 in its reasoning, that memory receives access_counter += 1 and importance += 5. After enough citations, it crosses the jump threshold and moves to the mid-term layer.
